1. Индексы к `user_id` и `product_id`:
   Влияние: повышает производительность запросов за счет ускорения поиска и объединения этих столбцов.
   Пример: при получении заказов для конкретного пользователя или продукта запрос будет выполняться быстрее.

2. Ограничения для незнакомого ключа:
   Влияние: обеспечивает целостность данных, предотвращая недопустимые записи `user_id` и `product_id` в таблице `user_order`.
   Пример: если кто-то попытается вставить запись в `user_order` с `user_id`, которого нет в таблице `user`, база данных отклонит вставку.

Применяя эти изменения, мы оптимизируем базу данных для повышения производительности и обеспечения целостности данных, что имеет решающее значение для поддержания надежного и эффективного приложения.

ALTER TABLE user_order ADD INDEX idx_product_id (product_id);

-- Добавить ограничения для незнакомого ключа
ALTER TABLE user_order 
ADD CONSTRAINT fk_user 
FOREIGN KEY (user_id) 
REFERENCES user(id);
